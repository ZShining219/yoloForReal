# Module Boundary

## What This Module Establishes

This module establishes a reviewed target-track layer for one real-video window:

```text
MVI_0866_520_560.mp4
```

The strongest defensible statement from this module is:

> YOLO11m + ByteTrack produced vehicle tracks; a coarse human pass accepted target recall as usable; a focused false-positive review removed 7 non-focus tracks; the final target table contains 38 reviewed target tracks; no vehicles were manually added.

This is a target-existence and target-trajectory claim. It is not an intersection semantics claim.

## Valid Downstream Use

Downstream modules may use:

```text
outputs/target_tracks/target_tracks_final.csv
outputs/target_tracks/detections_tracked.csv
```

to reconstruct candidate events, inspect trajectories, or debug whether a lane-mapping failure starts from target detection or from later geometry/semantics.

## Invalid Downstream Use

This module alone does not support claims that:

- a vehicle entered the intersection through a named lane such as `W2J_001`;
- a vehicle exited through a named road edge;
- a lane supports right/straight/left movements;
- a track crossed an annotated stop line or lane line;
- a target's movement is a specific turn relation such as south-to-east right turn;
- the generated SUMO route, demand, or timing is correct.

Those claims require separate lane geometry, lane function mapping, line-crossing logic, trajectory extension, movement prior, and timing audit modules.

## Debugging Implication

For the current bug triage, this module should be treated as the first checkpoint:

1. If the target is missing from `target_tracks_final.csv`, the issue belongs to target extraction or human filtering.
2. If the target exists in `target_tracks_final.csv` but receives a wrong entry/exit/lane/time label, the issue belongs downstream of this module.
3. If a downstream route uses any table other than `target_tracks_final.csv` as its target source, that is a reproducibility and boundary error.

## Current Quality Status

The packaged target-extraction layer is internally reproducible for this pass:

- `target_tracks_after_fp_filter.csv` can be regenerated byte-for-byte.
- `target_tracks_final.csv` can be regenerated byte-for-byte.
- `target_overall_review_gate.csv` can be regenerated byte-for-byte.

This does not mean the full real-video-to-SUMO result is qualified. The current intersection mapping and timing result still needs independent bug triage and redesign.
